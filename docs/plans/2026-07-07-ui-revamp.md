# 계획: UI 개편 (2026-07-07)

> 상태: 구현 완료(세션 2). 결정 기록은 ADR-0002, 진행 이력은 session-log 참고.

출처: 사용자 스크린샷(`스크린샷 2026-07-07 144053.png`) + 대화 피드백.

## 요구사항

1. **레이아웃 스왑**
   - 현재: 왼쪽 = 드롭존 + 파일목록 / 오른쪽 = 미리보기 + 변환옵션
   - 목표: 왼쪽 = 드롭존 + **미리보기(큰 영역)** / 오른쪽 = **파일목록** + 변환옵션
   - 사용자 표현: "미리보기가 [드롭존] 밑에 나왔으면, 첨부한 파일들은 지금 미리보기 자리에"

2. **대상 포맷별 옵션 창**
   - 이미지(PNG/JPEG/WebP)로 변환 시 → **가로 x 세로 픽셀** 입력으로 리사이즈.
     - 둘 다 입력: 그 크기로. 하나만 입력: 비율 유지. 둘 다 비움: 원본 크기.
   - PDF→이미지: 기존 해상도(scale) 옵션 유지.

3. **다중 페이지 미리보기 (페이지 넘김)**
   - 여러 이미지를 PDF로 변환 시, 미리보기에서 각 이미지를 페이지로 보고 이전/다음 이동.
   - PDF 파일을 올렸을 때도 페이지별로 넘겨보기(기존엔 1페이지만).

## 설계 결정

- 미리보기를 `components/Preview.tsx` 로 분리. 페이지네이션·렌더링·URL 수명관리를 캡슐화.
- `Preview` 는 `PreviewSource` 를 받음:
  - `{ type: 'images', urls: string[] }` — 이미지 object URL 배열(원본은 App 소유, Preview는 revoke 안 함).
  - `{ type: 'pdf', bytes, scale }` — pdf.js로 페이지 수 조회 + 현재 페이지만 지연 렌더(캐시, Preview 소유 → revoke).
- App이 상태에 따라 소스를 고름:
  - 이미지들 + 대상 PDF → `images`(전체 파일을 페이지로)  ← 요구사항 3의 핵심
  - 활성 파일이 이미지 → `images`(1장)
  - 활성 파일이 PDF → `pdf`
- 리사이즈: `convertImageFormat(bytes, fromMime, toKind, resize?)` 에 `{width?,height?}` 추가.
  디스패처 `runConversion` 의 `ConvertOptions.resize` 로 전달.

## 손대는 파일

- `convert/image.ts` — resize 인자
- `convert/index.ts` — ConvertOptions.resize 전달
- `components/Preview.tsx` — 신규(페이지네이션 미리보기)
- `App.tsx` — 레이아웃 스왑, resize 상태/입력, Preview 연결
- `styles.css` — 레이아웃, 미리보기 네비 스타일

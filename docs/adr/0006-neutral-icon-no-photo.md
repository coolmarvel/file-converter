# ADR-0006: 중립 아이콘 전환 & 개인 사진 사용 금지

## 상태

Accepted — ADR-0004의 "얼굴 사진 기반 아이콘/아바타" 부분을 **대체(supersede)** 한다.
(서명 sign.png 워터마크·크레딧, 이름 표기는 유지.)

## 맥락

사용자 지시(2026-07-09): "인스톨러부터 실행기까지 내 사진이 박힌 거 다 지우고 괜찮은
아이콘으로 교체. 깨지지 않고 부드러운 것. **앞으로 인스톨러든 실행기든 내 사진은 쓰지 않는다.**"

기존(v1.0.5까지)은 얼굴 사진이 앱 아이콘(icon.ico/png), NSIS 사이드바·헤더 BMP,
앱 헤더 로고, 푸터 아바타(face.png)에 들어가 있었다.

## 결정

1. **정책: 사용자 얼굴 사진을 어떤 산출물에도 넣지 않는다** (아이콘·인스톨러·앱 UI·문서 이미지).
   서명(sign.png)은 사진이 아니라 기능(서명 워터마크)·크레딧 요소이므로 유지.
2. 아이콘은 **코드로 그린 SVG**(라운드 사각형 파란 그라데이션 + 흰 문서 + 순환 화살표 2개)를
   `@resvg/resvg-js`로 래스터화. **각 ICO 크기(16~256)를 SVG에서 개별 렌더** → 다운스케일 아님,
   소형 크기에서도 또렷·부드러움. ICO 조립은 `png-to-ico`.
3. `scripts/gen-branding-assets.js` 전면 재작성 — 외부 사진 원본(`C:\...\이성현`) 의존 제거.
   임시 의존성: `npm i --no-save @resvg/resvg-js png-to-ico jimp@0.22`(jimp는 BMP 합성·텍스트용, v1 API 비호환 주의).
4. 산출물: `build/{icon.ico,icon.png,installer*.bmp,uninstaller*.bmp}`,
   `src/renderer/src/assets/app-icon.png`(앱 헤더 로고). `face.png` 삭제, 푸터 아바타 제거.

## 근거

| 대안 | 장점 | 단점 | 결론 |
|------|------|------|------|
| 무료 아이콘 다운로드 | 즉시 사용 | 라이선스 추적 필요, 재생성 불가, 해상도 고정 | 기각 |
| jimp 픽셀 스캔으로 도형 직접 그림 | 의존성 적음 | 안티앨리어싱 품질 낮음("깨짐") | 기각 |
| **SVG 작성 → resvg 래스터** | 벡터라 모든 크기 선명, 재생성·수정 쉬움 | 임시 의존성 2개 | **채택** |

## 결과

- v1.0.6부터 적용. 아이콘 디자인 수정 = 스크립트의 `buildIconSvg()`만 고치고 재실행.
- ADR-0004의 누끼/합성 로직은 폐기(스크립트에서 삭제됨). 이름·서명 크레딧은 계속 유지.

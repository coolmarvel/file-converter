# 변경 이력

인스톨러 배포 단위의 사람용 요약. 최신이 위. 개발 진행 상세는 `session-log.md` 참고.

## [1.3.1] — 2026-07-13

- **UI 전반 확대·개선**: 글자·아이콘·버튼·툴바가 전체적으로 한 단계 커짐 (작아 보인다는 피드백).
  랜딩 화면과 버튼 스타일 다듬음, 키보드 포커스 표시 추가.
- **앱 시작 속도 개선**: PDF 엔진을 실제로 PDF를 다룰 때만 불러오도록 분리 (시작 코드 60% 감량).
- 내부 코드 정리(중복 제거·공용 부품화) — 기능 변화 없음, v1.3.0 기능 전부 회귀 검증 완료.

## [1.3.0] — 2026-07-13

- **배경 제거** — 컨텍스트 툴바 [배경] 메뉴:
  - **흰색 → 투명**: 흰 배경을 투명하게 (허용 오차 슬라이더). 도장·서명·로고에 적합.
  - **AI 배경 제거**: 인물·사물 누끼 (완전 오프라인, AI 모델 내장). 처리에 장당 수 초 소요.
  - 미리보기에 즉시 반영되고 투명 영역은 체커보드로 표시. 투명도는 PNG·WebP·ICO 저장에서 유지.
- **지원 포맷 확장**: 입력에 **HEIC(아이폰 사진)·TIFF(스캔)·ICO** 추가, 출력에
  **ICO 아이콘(16~256 멀티사이즈)**·**SVG 벡터화(로고·단순 이미지용)** 추가.
- **PDF 문서 정리**: PDF를 열면 [전체 병합]과 [페이지 도구](분할·회전·페이지 삭제·순서 변경)를 바로 사용.
- **Ctrl+V 붙여넣기**: 클립보드의 스크린샷/이미지를 바로 파일로 추가.
- **진행 표시**: 오래 걸리는 작업(AI 처리, 여러 파일 변환 등)에 로딩바 표시.
- ⚠️ AI 모델 내장으로 인스톨러가 크게 커짐 (오프라인 동작 유지 목적, 사용자 승인).

## [1.2.0] — 2026-07-13

- **자르기(크롭)** — 그림판처럼 미리보기에서 드래그로 남길 영역을 지정. 바깥의 어두운 부분이
  잘려나갈 부분. 영역은 드래그로 이동, 모서리 핸들로 크기 조절, 컨텍스트 툴바 ×로 해제.
  회전·크기 조절과 함께 쓰면 **미리보기에 보이는 그대로** 잘림. PDF→이미지 변환에도 적용(전 페이지 동일 영역).
- **실행취소/다시실행** — 파일 추가·삭제·순서변경과 모든 옵션 변경(자르기·회전·워터마크 등)을
  거슬러 올라갈 수 있음. 툴바 버튼 또는 **Ctrl+Z / Ctrl+Y**.

## [1.1.4] — 2026-07-13

- **드래그&드롭으로 파일을 넣으면 같은 파일이 2장 추가되던 버그 수정** — 랜딩 드롭존과
  본문 전체 드롭 처리가 중복 실행되던 문제. 이제 끌어다 놓아도 1장씩만 추가됨.
  (파일 탐색기로 추가하는 경로는 원래 정상)

## [1.1.3] — 2026-07-13

- **미리보기 = 결과물 (보이는 그대로 저장)** — PDF 편집기처럼, 컨텍스트 툴바에서 고른 옵션이
  미리보기에 즉시 반영됨: **회전·좌우/상하 반전·흑백·출력 크기(비율)**. 변환 전에 눈으로 확인 가능.
- **워터마크 기본 진하기 22% → 35%** (PDF 편집기와 동일) — 사진 위에서 너무 연해
  "적용이 안 된 것처럼" 보이던 문제. 워터마크 자체는 정상 합성되고 있었음(진하기 슬라이더로 조절 가능).

## [1.1.2] — 2026-07-13

- **미리보기 100% = "화면에 맞춤"** — PDF 편집기의 레이아웃 → 화면에 맞춤과 동일한 동작.
  이미지든 PDF든 100%에서 **전체가 스크롤 없이 미리보기 화면 안에** 들어오고(가로·세로 중
  넘치는 쪽 기준), 화면 정중앙에 표시됨. 확대(+)하면 그 배율의 배수로 커지며 스크롤.
  "맞춤" 버튼 = 100% 복귀. (v1.1.1의 "폭만 맞춤"은 세로로 긴 사진이 화면 밖으로 넘치던 문제)

## [1.1.1] — 2026-07-13

- **DICOM 변환 제거** — DICOM은 별도 전용 변환기 앱으로 특화 개발 예정. 이 앱은 PDF·이미지 변환에 집중.
  (환자 정보 폼, DICOM 대상 버튼, dcmjs 라이브러리 모두 삭제)
- **지원 포맷 확장**:
  - 입력에 **BMP · GIF · SVG · AVIF** 추가 (GIF/SVG/AVIF는 읽기 전용 — 다른 포맷으로 변환만 가능).
  - 출력에 **BMP** 추가, PDF→이미지에 **WebP** 추가.
  - **같은 포맷 → 같은 포맷 변환 허용**: JPEG를 JPEG로 고르면 크기·품질만 바꿔 다시 저장 가능.
- **변환 옵션 확장** (컨텍스트 툴바):
  - **품질 슬라이더**(JPEG/WebP, 10~100%) — 낮출수록 파일이 작아짐.
  - **회전(90° 단위) · 좌우/상하 반전 · 흑백(그레이스케일)**.
  - 출력 크기(px)는 이미지→PDF에도 적용, PDF→이미지도 크기 지정 가능.
- **미리보기 100% = 화면 폭 맞춤 고정**: 이미지·PDF가 항상 미리보기 영역 폭에 딱 맞게 표시
  (이전에는 조건에 따라 원본 픽셀 크기로 나올 수 있었음). "맞춤" 버튼 = 100% 리셋.

## [1.1.0] — 2026-07-10

- **UI 전면 개편** — 자매 앱 "PDF 편집기"와 같은 디자인 시스템(MUI) 이식. 변환 기능·동작은 그대로.
  - 우측 사이드바에 몰려 있던 변환 옵션을 **상단 툴바 + 컨텍스트 툴바**로 재배치:
    상단 툴바에서 변환 대상(PDF/PNG/JPEG/WebP/DICOM)을 고르면, 바로 아래 컨텍스트 툴바에
    그 대상에 맞는 옵션(해상도·출력 크기·환자 정보·워터마크)만 나타남.
  - 파일 목록은 **좌측 사이드바**로 이동(접기 가능), 미리보기가 본문 전체를 사용.
  - 변환 실행은 우상단 **[N개 변환 후 저장]** 버튼, 결과 안내는 하단 토스트로 표시.
  - DICOM 환자 정보는 컨텍스트 툴바의 [환자 정보] 버튼 → 팝오버 폼으로 입력(필수 누락 시 경고 표시).
  - 브랜드 색은 기존 파랑(#3b74f2) 유지, 회색조·그림자·라운드 규칙은 PDF 편집기와 동일.

## [1.0.6] — 2026-07-09

## [1.0.4] — 2026-07-08 (사용자 확인 완료)

- **워터마크 켠 채 페이지 넘기면 꺼지던 문제 수정**: 워터마크 미리보기 캔버스가 확대 배율 × 윈도우 화면 배율(DPI)만큼
  무한정 커지던 것(400%+150% 배율이면 수백 MB)에 **해상도 상한**을 둠. 보이는 크기는 그대로.
- **페이지 넘김 방식 개선**: 넘기는 동안 이전 페이지를 계속 보여주고 작은 "불러오는 중…" 표시만 띄움
  (화면을 매번 부수고 다시 만들지 않아 더 안정적이고 덜 깜빡임).
- **자동 복구 안내**: 만에 하나 화면이 리셋되면 "자동 복구했습니다. 파일을 다시 추가해 주세요" 안내가 뜸
  (이전엔 조용히 리셋되어 앱이 꺼진 것처럼 보였음).

## [1.0.3] — 2026-07-08

- **대용량 PDF에서 페이지 넘기다 앱이 꺼지는 문제 수정** (30페이지 매뉴얼 등):
  - 페이지를 넘길 때마다 PDF 전체를 다시 읽던 구조를 → **문서를 한 번만 열고 페이지만 렌더**하도록 개편.
    넘김이 빨라지고, "다음"을 연타해도 메모리가 폭증하지 않음 (135MB·30페이지 테스트: 연타 시 1.4GB → 0.9GB 이내).
  - 스캔본처럼 페이지가 거대한 PDF도 미리보기 캔버스가 폭주하지 않게 해상도 상한 적용.
  - 만에 하나 화면 프로세스가 죽어도 **앱이 꺼지는 대신 자동으로 다시 로드**되는 안전망 추가(실제 강제 크래시 테스트로 확인).

## [1.0.2] — 2026-07-08 (설치 테스트 대기)

- **미리보기 확대 폭주 수정**: 확대하면 화면이 말도 안 되게 커지고 축소가 안 되던 문제의 진짜 원인
  (레이아웃이 내용 크기를 따라 무한히 넓어지는 되먹임)을 잡음. 이번엔 Playwright 자동 테스트로
  확대(140%·200%)/축소(40%)와 크기 수치를 실제 브라우저에서 검증함.
- **워터마크 미리보기 표시 정상화**: 같은 폭주 버그 때문에 오버레이 캔버스가 메모리 초과로 그려지지
  못하던 것 — 수정 후 대각선/바둑판 배치·확대 상태에서 실시간 표시되는 것 확인(스크린샷 검증).
